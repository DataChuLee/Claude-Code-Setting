# /review — 코드 리뷰

**사용법**: `/review` 또는 `/review {파일명 또는 기능명}`

## 실행 순서

1. 변경된 파일 또는 지정 파일을 **[reviewer agent]**에 전달
2. 품질, 보안, 설계 관점에서 리뷰
3. 리뷰 결과 출력 (🔴 필수 수정 / 🟡 권장 / 💡 제안)
4. 리뷰 결과를 해당 기능 폴더의 `TODO.md` 리뷰 기록 섹션에 추가
   - 🔴 필수 수정 항목 → NOW 섹션에 추가
   - 🟡 권장/💡 제안 항목 → 리뷰 기록 섹션에 날짜와 함께 기록
5. 필수 수정 사항이 있으면 수정 후 `/review` 재실행

## 리뷰 대상
- 인자 없이 실행: 마지막으로 수정된 파일들 자동 감지
- 파일 지정: `/review src/auth/service.py`
- 기능 지정: `/review 로그인 기능`

## 예시
```
/review
/review auth/
/review JWT 인증 구현
```
