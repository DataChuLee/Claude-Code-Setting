# /tdd — TDD 전체 루프

**사용법**: `/tdd "구현할 기능명"`

## 실행 순서

1. **[폴더 초기화]** `{기능명}/` 폴더가 없으면 생성
   - `{기능명}/CLAUDE.md` 생성 — 기능 목적/역할, API 엔드포인트, 의존성, 기술 결정 사항 초안 작성
   - `{기능명}/TODO.md` 생성 — NOW 섹션에 구현할 항목 목록 작성
2. **[builder agent]** 테스트 케이스 설계 + `tests/test_{기능명}.py` 작성
3. **[PostToolUse Hook]** 파일 저장 감지 → `pytest` 자동 실행 → 실패 확인 (Red)
4. **[구현]** 테스트를 통과하는 최소 코드 작성
5. **[PostToolUse Hook]** 저장 → `pytest` 재실행 → 통과 확인 (Green)
6. **[TODO 업데이트]** Green 확인 후 `{기능명}/TODO.md`의 NOW 항목을 DONE으로 이동
7. **[리팩터]** 코드 정리 (테스트 유지)
8. **[Stop Hook]** `ruff check --fix . && ruff format .` 자동 실행
9. `/commit` 으로 커밋

## 주의사항
- 한 번에 하나의 기능만 TDD
- 테스트가 통과하기 전에 다음 기능으로 넘어가지 말 것
- 구현이 막히면 테스트를 더 작은 단위로 쪼갤 것
- `{기능명}/CLAUDE.md`의 API 엔드포인트/의존성은 구현하면서 실제 내용으로 채울 것

## 예시
```
/tdd "사용자 로그인"
/tdd "장바구니 총액 계산"
/tdd "JWT 토큰 갱신"
```
